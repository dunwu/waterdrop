- 关注和试点 SkyWalking。

用好调用链监控，需要订制化、自研能力。

## 典型应用场景

- **技术选型决策**：根据团队规模、技术栈、预算等因素，对比 CAT、Zipkin、SkyWalking 等工具的特性，选择最适合的监控方案。
- **监控体系规划**：结合 APM、日志、指标三大支柱，规划完整的可观测性体系，避免工具重叠或覆盖不全。
- **成本评估**：评估不同监控工具的部署成本、运维成本、学习成本，在功能需求和资源投入间取得平衡。
- **演进路径设计**：从单一监控工具起步，逐步演进到全方位可观测性平台，制定合理的迁移和升级计划。

## 最佳实践

- **明确监控目标**：在选型前明确核心需求（如链路追踪、实时告警、业务指标），避免被功能列表迷惑而选择过于复杂的工具。
- **试点验证**：在非核心系统先试点，验证工具的性能、稳定性、易用性，再决定是否全公司推广。
- **考虑生态兼容**：选择与现有技术栈（Spring Cloud、Dubbo、K8s 等）兼容性好的工具，降低集成成本。
- **预留扩展空间**：选择支持插件化、可扩展的工具，为未来新增监控维度（如 Service Mesh、eBPF）预留空间。

## 常见问题

**CAT、Zipkin、SkyWalking 的核心差异是什么？**

CAT 是实时监控系统，强在丰富的报表和告警能力，适合企业级应用；Zipkin 专注分布式追踪，轻量易用，适合中小规模；SkyWalking 是完整的 APM 平台，提供追踪、指标、日志一体化方案，适合大型微服务架构。

**如何避免监控工具成为性能瓶颈？**

1) 合理设置采样率，非核心接口降低采集比例；2) 使用异步批量上报，避免阻塞业务线程；3) 监控服务端独立部署，与业务系统资源隔离；4) 定期评估监控工具本身的资源消耗，及时优化或扩容。
