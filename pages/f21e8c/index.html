# 分布式

大型软件为了应对海量数据、高并发，一般都会被设计为分布式系统。

分布式系统需要解决很多不同领域的问题。

# 分布式理论综合资料

  • 教程
    • MIT-6.824 (opens new window) - 麻省理工分布式系统课程
    • CMU-15-440 (opens new window) - 卡内基梅隆分布式系统课程
    • Standford-CS244b (opens new window) - 斯坦福分布式系统课程
    • UC Berkley-CS294-91 (opens new window)- 伯克利分布式计算课程
    • 分布式技术原理与算法解析 (opens new window) - 极客时间教程
    • 分布式协议与算法实战 (opens new window) - 极客时间教程
  • 书籍
    • 分布式系统原理与范型 (opens new window)：书原名 Distributed Systems Principles and Paradigms。经典分布式教程，介绍了分布式系统的七大核心原理，并给出了大量的例子；系统讲述了分布式系统的概念和技术，包括通信、进程、命名、同步化、一致性和复制、容错以及安全等。
  • 文章
    • Solution of a Problem in Concurrent Programming Control (opens new window)，译文 (opens new window) - Dijkstra 首次提出并解决了互斥执行问题
    • Time, Clocks, and the Ordering of Events in a Distributed System (opens new window)，译文 (opens new window)，解读 (opens new window) - Lamport 介绍 happened before、偏序关系（partial ordering）、逻辑时钟（Logical Clocks）概念，提出解决分布式系统中区分事件发生的时序问题的方法。
    • Virtual Time and Global States of Distributed Systems (opens new window)，解读 (opens new window) - 逻辑时钟无法描述事件的因果关系。本文提出了向量时钟，这种算法利用了向量这种数据结构将全局各个进程的逻辑时间戳广播给各个进程，通过向量时间戳就能够比较任意两个事件的因果关系。
    • Brewer’s Conjecture and the Feasibility of Consistent, Available, Partition-Tolerant Web Services (opens new window)，解读 (opens new window) - 经典的 CAP 理论，即：在一个分布式系统中，当发生网络分区时，那么强一致性和可用性只能二选一。
    • CAP Twelve Years Later: How the “Rules” Have Changed (opens new window) - CAP 理论的新解读，并阐述 CAP 理论的一些常见误区。
    • BASE: An Acid Alternative (opens new window)，译文 (opens new window) - BASE 理论是对 CAP 中一致性和可用性的权衡，提出采用适当的方式来使系统达到最终一致性。
    • A Simple Totally Ordered Broadcast Protocol (opens new window) - 概述 ZooKeeper 的全序广播协议（Zab）
    • The Eight Fallacies of Distributed Computing - Tech Talk (opens new window) - 分布式系统新手常犯的 8 个错误，并探讨了其会带来的影响。
    • Distributed Systems for Fun and Profit (opens new window) - 一本学习小册，涵盖了分布式系统中的关键问题，包括时间的作用和不同的复制策略。
    • A Note on Distributed Systems (opens new window) - 这是一篇经典的论文，讲述了为什么在分布式系统中，远程交互不能像本地对象那样进行。
    • 深度探索分布式理论经典论文 (opens new window) - 分布式理论论文的导读文章
    • Distributed Systems for fun and profit (opens new window)：分为五章，讲述了扩展性、可用性、性能和容错等基础知识，FLP 不可能性和 CAP 定理，探讨了大量的一致性模型；讨论了时间和顺序，及时钟的各种用法。随后，探讨了复制问题，如何防止差异，以及如何接受差异。此外，每章末尾都给出了针对本章内容的扩展阅读资源列表，这些资料是对本书内容的很好补充。
    • An introduction to distributed systems (opens new window) - 这是一份分布式系统的提纲挈领的介绍，几乎涵盖了所有知识点，并辅以简洁并切中要害的说明文字，适合初学者了解知识全貌，快速与现有知识结合，形成知识体系。
