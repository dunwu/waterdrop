Debugging

快捷键介绍

FAQ

（1）运行时报错

Error running XXX. Command line is too long. Shorten the command line via JAR manifest or via a classpath file and rerun

解决方案：

找到 .idea/libraies/workspace.xml 中的 <component name="PropertiesComponent">

添加一行配置：

<property name="dynamic.classpath" value="true" />

典型应用场景

  • Java/Spring 项目开发：IDEA 对 Spring、SpringBoot、MyBatis 等框架支持极佳，是 Java 后端开发的首选 IDE。
  • 微服务开发：配合 Spring Cloud、Docker 插件，支持微服务项目的开发、调试和部署。
  • 数据库操作：内置 Database 工具，可直接连接数据库、执行 SQL、查看表结构，替代 Navicat 等工具。
  • 代码审查与重构：强大的代码分析和重构功能，支持 Extract Method、Rename、Move 等多种重构操作。

最佳实践

  • 善用 Live Template：配置常用代码模板（如 sout、psvm、try-with-resources），大幅提升编码效率。
  • 配置统一编码和换行符：在 Settings > Editor > File Encodings 中统一设置 UTF-8，在 Code Style 中统一换行符。
  • 使用 Local History 恢复代码：IDEA 的 Local History 会记录本地文件的每次修改，即使未提交 Git 也能找回丢失的代码。
  • 善用 Alt+Enter 快速修复：当代码有红色波浪线时，Alt+Enter 提供快速修复建议（导入包、处理异常、类型转换等）。

常见问题

IDEA 内存不足、卡顿？

修改 idea.vmoptions 文件，增大堆内存：-Xms512m -Xmx2048m。同时关闭不需要的插件和索引目录。

编译报 "Command line is too long"？

在 .idea/workspace.xml 的 PropertiesComponent 中添加 <property name="dynamic.classpath" value="true" />，或在 Run Configuration 中设置 "Shorten command line" 为 JAR manifest。

如何恢复误删的文件？

右键项目 > Local History > Show History，可以找到并恢复之前版本的文件。Local History 默认保疙5 天的记录。
